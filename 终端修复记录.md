# Claude Code 终端修复记录

## 故障现象

在 VS Code 中使用 Claude Code 扩展时，所有需要执行命令的操作都失败，返回 `exit code 1` 且没有任何输出。即使是最简单的命令（如 `echo hello`、`whoami`）也无法执行。

## 排查过程

| 步骤 | 操作 | 结果 | 结论 |
|------|------|------|------|
| 1 | 执行基础命令 `echo hello` | 失败，exit code 1 | 确认终端不可用 |
| 2 | 查询 PowerShell 执行策略 `Get-ExecutionPolicy` | 返回 `Bypass` | 排除执行策略问题 |
| 3 | 在 VS Code 内置终端手动输入 `echo "test from terminal"` | 正常输出 | 排除系统环境问题，问题锁定在 Claude Code 扩展本身 |
| 4 | 运行 `claude --version` | 2.1.141，正常 | 排除 CLI 问题 |
| 5 | 检查 `~/.claude/settings.json` 配置 | 无异常配置 | 排除配置文件问题 |
| 6 | 检查安全软件、重装扩展、禁用沙箱 | 都无效 | 排除常规原因 |
| 7 | 查看 Claude Code 日志 `Claude VSCode.log` | 发现每一条失败日志都是：`PowerShell parser: pwsh exited 1: 乱码 (9ms)` | 找到关键线索 |
| 8 | 发现 pwsh 来自 Store 版路径 `C:\Users\zhoulinhong\AppData\Local\Microsoft\WindowsApps\pwsh.exe` | — | **锁定根因** |
| 9 | 下载安装 MSI 版 PowerShell 7（非 Store 版），关闭 Store 版别名，将 MSI 版路径加入 PATH | — | 修复完成 |

## 根本原因

Microsoft Store 版本的 PowerShell 7 通过 Windows App Execution Alias 运行。当 Claude Code 的原生二进制文件 `claude.exe` 尝试调用 Store 版 pwsh 时，被 Windows 安全沙箱拦截，导致 pwsh 在 9 毫秒内直接退出，无法执行任何命令。

## 解决方法

### 1. 安装 MSI 版 PowerShell 7

从 GitHub 下载安装：https://github.com/PowerShell/PowerShell/releases

```powershell
# 下载
Invoke-WebRequest -Uri "https://github.com/PowerShell/PowerShell/releases/download/v7.6.1/PowerShell-7.6.1-win-x64.msi" -OutFile "D:\pwsh.msi"

# 安装（需管理员权限，会弹出安装向导）
Start-Process msiexec.exe -ArgumentList '/i "D:\pwsh.msi"' -Verb RunAs -Wait
```

建议安装到 D 盘路径（如 `D:\PowerShell\7\`）。

### 2. 关闭 Store 版别名

- 按 `Win + I` 打开设置 → 搜索 "应用执行别名" → 关闭 "PowerShell 7 (pwsh)"

### 3. 将 MSI 版加入 PATH

```powershell
[Environment]::SetEnvironmentVariable("PATH", "D:\PowerShell\7\7;" + [Environment]::GetEnvironmentVariable("PATH"), "User")
```

### 4. 重启 VS Code

完全关闭 VS Code，重新打开项目。
