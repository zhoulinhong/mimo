## 二、具体学习内容

### 2.1 Linux（CentOS 7）

#### 必须熟练（天天用）

- 文件与目录：`ls`, `cd`, `cp`, `mv`, `rm`, `find`, `tar`, `scp`
- 文件查看与编辑：`cat`, `less`, `tail -f`, `grep`, `vim`（打开、编辑、保存、退出）
- 权限管理：`chmod`, `chown`, 看懂 `ls -l` 每一列
- 用户与组：`useradd`, `usermod`, `passwd`, `/etc/passwd`, `/etc/group`
- 进程管理：`ps aux`, `top`, `kill`, `nohup &`, `systemctl`
- 磁盘与存储：`df -h`, `du -sh`, `lsblk`, `fdisk -l`, `mount`, `/etc/fstab`
- 网络基础：`ip addr`, `ss -tlnp`, `ping`, `curl`
- 软件安装：`yum` 包管理，源码编译安装（`./configure && make && make install`）

#### 数据库相关工作重点

- systemd 服务管理：`systemctl start/stop/status/enable`，会写 `.service` 文件
- 日志查看：`journalctl`, `/var/log/messages`, `tail -f` 实时看日志
- Shell 脚本：变量、`if`、`for`、函数、`crontab` 定时任务
- LVM 逻辑卷管理：`pvcreate`, `vgcreate`, `lvcreate`, `lvextend`
- 防火墙：`firewall-cmd` 开放端口
- ulimit 调优：`/etc/security/limits.conf`
- 内核参数：`/etc/sysctl.conf`, `sysctl -p`

#### 存储与 IO（沃趣核心方向）

- IO 性能测试：`fio` 测磁盘 IOPS 和延迟
- 文件系统：ext4 vs XFS 区别，挂载选项 `noatime`
- RAID 概念：RAID 0/1/5/10
- NVMe/SSD 基础：`nvme list`

#### 网络

- TCP/IP 基础：三次握手、四次挥手
- 抓包：`tcpdump` 基本用法
- SSH 免密登录：`ssh-keygen` + `ssh-copy-id`

#### 了解即可

- Docker 基础：`docker run`, `docker exec`, `docker-compose`
- K8s 概念：Pod / Service / Deployment / Namespace
- Git：`clone`, `commit`, `push`, `pull`, `branch`, `merge`

#### Linux 检验标准

> 找一台裸 Linux 服务器，从零安装 MySQL，调整内核参数和防火墙，挂载数据盘，写 systemd 保证开机自启，写 shell 脚本每天凌晨自动备份并保留最近 7 天备份。

### 2.2 数据库操作（MySQL / PostgreSQL）

#### 基础篇

- 建库、建表、插入数据
- JOIN 查询、子查询
- 索引：加索引，用 `EXPLAIN` 看加索引前后查询计划变化
- 往表里插入 100 万行数据，感受性能差异
- `mysqldump` 备份全库，恢复到另一个库
- 慢查询日志：找出执行超过 1 秒的 SQL，优化它

#### 进阶篇

- MySQL 主从复制：搭一套主从，验证主库写入后从库能读到
- 误删恢复：模拟误删 → 用备份 + binlog 恢复
- 调参压测：调 `my.cnf`（buffer pool size 等），用 sysbench 压测看效果

#### 数据库检验标准

> 独立完成 MySQL 主从复制搭建，写 shell 脚本自动备份，能用 EXPLAIN 分析并优化慢查询。

## 三、学习方法

### 核心原则：场景驱动，不要背命令

**错误做法**：对着命令列表一个个敲，像背单词一样记参数。

**正确做法**：先想"我要干什么"，再查怎么干。同一个场景反复用 3-5 次自然记住。

脑子里存的不是命令，是映射：
> "看磁盘剩余" → `df -h`
> "看目录占多大" → `du -sh *`
> "实时看日志" → `tail -f`
> "看 3306 端口在不在监听" → `ss -tlnp | grep 3306`

### 练习方式：在自己环境里实战

1. **VMware CentOS 7 虚拟机**就是练兵场，所有操作在终端里完成
2. 用 Docker 拉 MySQL 容器作为练习目标，随便折腾不心疼
3. 去 Kaggle 下载真实数据集导入练习，比虚构数据有效
4. 从 GitHub 找开源项目的数据库 schema，学习别人的表设计

### 查资料的优先级

1. `man 命令名` 或 `命令 --help`
2. 官方文档（MySQL、PostgreSQL 官方文档质量很高）
3. Stack Overflow（英文搜索）
4. 问 AI（ChatGPT / Claude）

**不要用 CSDN 和中文博客**，错误多、过时多、互相抄。
